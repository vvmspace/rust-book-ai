# Пути к элементам в дереве модулей

Чтобы найти функцию, нужно знать её адрес (путь).
Пути бывают двух видов:

1.  **Абсолютный:** начинается с корня крейта (`crate::...`).
2.  **Относительный:** начинается с текущего модуля (`self::`, `super::` или просто имя).

```rust
mod front_of_house {
    pub mod hosting {
        pub fn add_to_waitlist() {}
    }
}

pub fn eat_at_restaurant() {
    // Абсолютный путь
    crate::front_of_house::hosting::add_to_waitlist();

    // Относительный путь
    front_of_house::hosting::add_to_waitlist();
}
```

Мы предпочитаем абсолютные пути, потому что код определений и код вызовов часто перемещаются независимо друг от друга.

### `super`: Привет, сосед сверху

`super` — это как `..` в файловой системе. Позволяет обратиться к родительскому модулю.

```rust
fn serve_order() {}

mod back_of_house {
    fn fix_incorrect_order() {
        cook_order();
        super::serve_order(); // Вызываем функцию из родителя
    }
    fn cook_order() {}
}
```

### Приватность структур и энамов

Если структура `pub`, её поля **по умолчанию приватные**. Нужно ставить `pub` перед каждым полем, которое вы хотите открыть.
Если энам `pub`, то **все его варианты публичные**. Логично, да? Бесполезно иметь энам, но не знать его варианты.
