# `Rc<T>`: Умный указатель с подсчетом ссылок

Обычно владение в Rust единоличное. Но иногда нужно, чтобы у значения было несколько владельцев.
Например, в графе у узла может быть несколько входящих ребер.

`Rc<T>` (Reference Counting) ведет подсчет ссылок. Данные удаляются только когда счетчик равен 0.

### Аналогия с телевизором
Представьте телевизор в гостиной. Первый вошедший включает его. Остальные приходят и смотрят.
Телевизор выключается только тогда, когда **последний** человек выходит из комнаты.

### Использование

```rust
use std::rc::Rc;
use crate::List::{Cons, Nil};

enum List {
    Cons(i32, Rc<List>),
    Nil,
}

fn main() {
    let a = Rc::new(Cons(5, Rc::new(Cons(10, Rc::new(Nil)))));
    // Rc::clone увеличивает счетчик, но НЕ копирует данные глубоко
    let b = Cons(3, Rc::clone(&a)); 
    let c = Cons(4, Rc::clone(&a));
}
```

`Rc::clone(&a)` — это очень быстрая операция (просто инкремент счетчика).

**Важно:** `Rc<T>` работает **только** в однопоточных сценариях. И он дает только иммутабельный доступ.

---

**English Joke:**

> How many reference counters does it take to change a lightbulb?
> One, but only if the count reaches zero.
