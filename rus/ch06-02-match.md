# Управляющая конструкция `match`

`match` — это как `switch` в других языках, но на стероидах, с IQ 200 и черным поясом по карате.
Он сравнивает значение с набором шаблонов. Если шаблон совпал — выполняется код.

```rust
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}
```

Это похоже на сортировщик монет. Монета катится и падает в первую подходящую дырку.

### Привязка к значениям

Шаблоны могут доставать значения *изнутри* вариантов.

```rust
enum Coin {
    Quarter(UsState),
    // ...
}

match coin {
    Coin::Quarter(state) => {
        println!("Quarter from {:?}!", state);
        25
    },
    _ => 0, // Остальное нас не волнует
}
```

### Сопоставление с `Option<T>`

`match` идеален для `Option`.

```rust
fn plus_one(x: Option<i32>) -> Option<i32> {
    match x {
        None => None,
        Some(i) => Some(i + 1),
    }
}
```

Представьте, что `match` — это фейс-контроль.
"Ты `None`? Проходи мимо, ничего не делай. Ты `Some(i)`? О, давай сюда своё `i`, мы добавим единичку и упакуем обратно."

### Исчерпывающий (Exhaustive) контроль

Rust не даст вам забыть.
Если вы не обработаете *хотя бы один* вариант энама, код не скомпилируется.

```rust
match x {
    Some(i) => println!("{}", i),
    // Ошибка! А что делать с None?
}
```

Это спасает от багов, когда вы добавляете новый вариант в энам, но забываете обновить switch-кейсы по всему коду.

### Заполнитель `_`

Если вам интересен только один случай, а на остальные плевать:
```rust
let dice_roll = 9;
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    _ => reroll(), // Для всего остального
}
```
