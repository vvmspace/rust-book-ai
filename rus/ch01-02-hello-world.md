# Hello, World!

Традиции нарушать нельзя. Пишем первую программу.

### Создаем проект

Создайте папку `projects`, а в ней — `hello_world`.
Заходим туда.

### Пишем код

Создайте файл `main.rs`. Обратите внимание на расширение: `.rs`.
Если имя файла из нескольких слов, используйте `snake_case` (например, `hello_world.rs`).

В файл пишем:

```rust
fn main() {
    println!("Hello, world!");
}
```

### Разбор полетов

1. `fn main() { ... }`: Функция `main`. Входная точка в любую программу.
2. `println!("...")`: Макрос для печати в консоль.
   - **Важно**: Видите `!`? Это значит, что это **макрос**, а не функция. В C++ макросы — это зло, а в Rust — суперсила.
3. `;`: Точка с запятой. Означает конец инструкции. Не забывайте их, а то компилятор обидится.

### Компиляция и запуск

В терминале:

```console
$ rustc main.rs
$ ./main  # или .\main.exe на Windows
Hello, world!
```

Поздравляю! Вы официально написали код на Rust. Теперь можно обновлять резюме.

Но компилировать вручную через `rustc` — это как забивать гвозди микроскопом. Дальше мы узнаем про Cargo.

---

**Анекдот:**

> Программист на Rust пишет "Hello world":  
> 1. `cargo new hello_world`  
> 2. `cargo run`  
> 3. Ждет, пока скачаются пол-интернета зависимостей (шутка, тут их нет).
>
> Программист на C++ пишет "Hello world":  
> 1. `#include <iostream>`  
> 2. `using namespace std;`  
> 3. Получает 3 страницы ошибок линковщика.
