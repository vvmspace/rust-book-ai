# Обработка ошибок

Ошибки — это факт жизни. Как и смерть, налоги и дедлайны.
В Rust есть два вида ошибок:

1.  **Поправимые (Recoverable):** Например, файл не найден. Мы можем попросить пользователя указать другой путь. Для этого используется `Result<T, E>`.
2.  **Непоправимые (Unrecoverable):** Баги. Например, выход за границы массива. Тут программа просто падает с помощью `panic!`.

В Rust нет исключений (exceptions).
Вместо того чтобы кидать исключения и надеяться, что кто-то их поймает, Rust заставляет вас обрабатывать возможные ошибки *явно*. Это делает код надежным, как швейцарский нож.

В этой главе мы научимся паниковать (правильно) и возвращать результаты.

---

**Анекдот:**

> Программист на Rust не совершает ошибок.
> Он просто возвращает `Err` и заставляет *вас* разбираться с этим.
