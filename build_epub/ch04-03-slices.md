# Слайсы (Slices)

Слайсы — это еще один вид ссылок, которые не владеют данными.
Слайс ссылается на **часть** последовательности элементов.

### Строковые слайсы

Представьте, что вам нужно первое слово из строки.

```rust
let s = String::from("hello world");
let hello = &s[0..5];
let world = &s[6..11];
```

`&s[0..5]` — это слайс. Внутри он хранит:

1. Указатель на начало (байт 0).
2. Длину (5).

Тип строкового слайса — `&str`.
Кстати, строковые литералы — это тоже слайсы:
```rust
let s = "Hello, world!"; // тип s: &str
```
Потому что они ссылаются на "зашитую" в бинарник константу.

### Зачем это нужно?

Допустим, у вас есть функция `first_word`, которая возвращает индекс пробела.

```rust
let mut s = String::from("hello world");
let word_index = first_word(&s); // возвращает 5

s.clear(); // s теперь пустая строка ""

// word_index все еще 5, но строки уже нет!
// Если мы попробуем использовать index 5 с пустой строкой s — будет баг или паника.
```

Слайсы решают эту проблему. Если `first_word` вернет `&str` (слайс строки), то он будет привязан к исходной строке.

```rust
fn first_word(s: &String) -> &str { ... }

let mut s = String::from("hello world");
let word = first_word(&s);

s.clear(); // ОШИБКА!
// s.clear() требует &mut s.
// word держит неизменяемую ссылку на часть s.
// Правило borrowing запрещает &mut, пока есть активные &.
// Rust спас нас от бага!
```

### Другие слайсы

Бывают не только строковые.

```rust
let a = [1, 2, 3, 4, 5];
let slice = &a[1..3]; // тип &[i32], содержит [2, 3]
```

Они работают так же: указатель + длина.

Это мощный инструмент, чтобы передавать части массивов, векторов и строк без копирования данных.
