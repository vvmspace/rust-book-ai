# Лаконичный `if let`

Иногда `match` — это слишком много букв.
Если вам нужен только *один* вариант, а остальные идут лесом, используйте `if let`.

Вместо:
```rust
let config_max = Some(3u8);
match config_max {
    Some(max) => println!("Max is {}", max),
    _ => (),
}
```

Пишем:
```rust
if let Some(max) = config_max {
    println!("Max is {}", max);
}
```

Это синтаксический сахар.
Он читается как: "Если `config_max` подходит под шаблон `Some(max)`, то выполни блок".

### `let...else`

А еще есть модный `let...else`, чтобы оставаться на "счастливом пути" (happy path) и не плодить вложенность.

```rust
let Some(max) = config_max else {
    return; // Обязан выйти из функции
};
// Теперь max доступен здесь, как обычная переменная
println!("We have a max: {}", max);
```

Это супер удобно для проверок в начале функции.

### Итоги главы

Мы узнали:

- `Enum` — мощный инструмент для "одного из".
- `Option` спасает от null-апокалипсиса.
- `match` заставляет обрабатывать все случаи.
- `if let` помогает писать короче.

Теперь вы готовы структурировать свои данные как боженька. В следующей главе мы научимся все это раскладывать по модулям, чтобы не утонуть в одном файле.
